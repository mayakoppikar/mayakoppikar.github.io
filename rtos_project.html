RTOS for Formula 1 Robot

This project focuses on building a real-time operating system (RTOS) for a Formula 1 robot. The RTOS manages motor control, sensor input, and inter-task communication with strict timing constraints.

Key Features
  • Task scheduling with priority-based preemption
  • Motor control via PWM
  • Sensor fusion from LIDARs
  • Interfacing via CAN bus
Tech Stack
  • Programming: C, ARM Assembly
  • Hardware: TM4C Microcontroller
  • RTOS Concepts: Scheduling, Semaphores, Queues

Major Challenges & Key Takeaways
  • OS Kernel Bugs: Our scheduler faced many kernel bugs throughout the process. Some of the most notable (and hardest to debug) were leaving OS Sleep and Kill functions within interrupts, trying to branch to a null address pointer when all tasks are sleeping, and conetxt switching within an interrupt instead of after exiting the interrupt handler.
  • Motor PWM: Towards the end, after we implemnted differential steering, we often ran into cases where the MCU would reset and the code would enter the Hard Fault Handler. The reason for this is that when we set our motor PWM duty cycle above 80-90%, this would draw too much current and cause the MCU to reset. We fixed this by capping our duty cycle to 80%.
  • Key Takeaways + Skils:
    • Using logic analyzers and other minimally intrusive debug techniques
    • Reading the disassembly window + understanding compiler optimizations
    • Reading sensor datasheets to write drivers!
